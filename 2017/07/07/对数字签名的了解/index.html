对数字签名的理解

以前对“数字签名”和“数字证书”的了解总是迷迷糊糊的，今天读了这篇，感觉有所了解。

RSA　加密（非对称加密）

公匙加密后的数据只能有私匙解密

私匙加密后的数据公匙都可以解密

eg（对一份文档的加密、传输过程）：

  1. A 要写一份文档发送给 B；
  2. A 写好文档之后，用 hash 函数计算出文档的摘要，然后将摘要用私匙加密，加密后的摘要就可以叫做数字签名，然后 A 将文档加上数字签名发送给 B；
  3. B 收到文件后，首先用 A 的公匙去解密数字签名，如果解密后的摘要和用 Hash 函数计算出来的摘要一致，说明文档没有被修改过。

中间人攻击

上面的 eg 有一个问题：如果 C 截获了 A 的请求，然后 C 将 C 的公匙、公匙加密后的摘要发送给 A，A 就会误以为是这个文档是 B 发送给他的。这个也就是 CA 出现的作用。

数字签名

加密后的摘要就可以叫做数字签名

数字证书

对A的公钥和一些相关信息一起加密，生成”数字证书”（Digital Certificate）。

CA（权威中心）

作用：对A的公匙进行认证，CA 用自己的私匙对 A 的公匙进行加密，B 拿到数字证书用 CA 的公司解密就可以活得 A 的公匙。

具体流程是：

  1. A 把文档、加密后的摘要、数字证书，发送给 B；
  2. B 收到文件后，用 CA 的公匙将数字证书解密，获得 A 的公匙；
  3. 然后 B 用 A 的公匙将摘要解密，然后用Hash函数计算出的摘要和解密后的摘要对比。

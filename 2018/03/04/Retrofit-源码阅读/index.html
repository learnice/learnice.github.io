Retrofit 的基本使用

构建 Retorfit 对象，先看看 Retrofit.Builder 的构造函数 (Builder 模式)

Retrofit.Builder.build() 最终构建出 Retrofit 对象

再看看 retrofit.create(GitHubService.class) 做了什么

返回了一个动态代理对象，接着看 invoke() 方法，loadServiceMethod(method) 用来给当前调用的方法生成 ServiceMethod 对象，这个对象用于将 Java 接口中定义的方法，转换成 HTTP 请求的相关参数

ServiceMethod 又是一个 Builder 模式，贴一下它的 build() 方法

重点看一下第一行 callAdapter = createCallAdapter();

最终会调用 retrofit.callAdapter(returnType, annotations);

其实这里的返回的 CallAdapter 就是在上面 retrofit 构建的时候添加的那个调用适配器 ExecutorCallAdapterFactory.get() 方法生成的，看看它做了什么

回到 Retrofit.creat() 方法，ServiceMethod 生成了，然后会生成 OkHttpCall ，最后调用 serviceMethod.callAdapter.adapt(okHttpCall)，然后返回生成的 ExecutorCallbackCall 对象(就是上面代码的 adapt 方法)

ExecutorCallbackCall 这里面用了一个代理机制，比如调用它的 enqueue() 方法，

OkHttpCall.enqueue() 方法

可以看到首先构建一个 okhttp3.Call 然后进行网络请求、数据解析、回调

总结

说说这几个类的作用

Retrofit： 整个框架的主体

ServiceMethod: 它就是将用户定义的接口方法，转化为真正的 Http 请求的各种参数，就是构建 OkHttp 请求

CallAdapter:

OkHttpCall: 实现了 Call，真正的调用 OkHttp 进行网络请求

Callback: 用来回调

Platform: 根据不同的平台（Android、Java），执行不同的策略
